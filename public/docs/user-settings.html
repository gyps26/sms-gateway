User Settings

📟 SIM

Manage how messages are handled when sending through your device’s SIM card.

  • Wait For Confirmation: When enabled, the app waits for confirmation from the mobile network operator before sending the next message. Disabling this can speed up sending but may increase the chance of failures.

🔄 Auto Retry

Configures auto retry functionally for a message campaign.

  • Enabled: Automatic retries will occur only if this option is enabled.

  • Max: The maximum number of retries the system will attempt. It will stop retrying once this limit is reached.

  • Change SIM or Sender ID After: You can disable this by unchecking the checkbox. Its value should be less than the max retries. When enabled, it will change the original SIM or Sender ID used to send the message with the most successful SIM or Sender ID in the campaign. This change will occur after a message was retried more than the number you set here. For example, if you set its value to 2, then it will only change it if a message still fails after the second retry. That means on the third retry it will try it with other SIM or Sender ID from the campaign.

📶 Dashboard

Settings to configure which widgets should be displayed on the dashboard.

  • Campaigns: If enabled, it will show stats for all campaign statuses.

  • Calls: If enabled, it will show stats for all call types.

  • Messages: If enabled, it will show stats for all message statuses.

  • USSD Pulls: If enabled, it will show stats for all USSD Pull statuses.

📧 Features

Settings to toggle various features across the system.

  • Message to Email: If enabled, it will forward received messages to the user's email. Users can only enable this if it is enabled by Admin.

  • Email to Message: If enabled, it will use the user's SIM or Sender ID to send a message containing the email body and any attachments to the phone number specified in the email subject. You can use the following formats for an email subject:

    • +1234567890 – This will send a message to +1234567890.

    • MMS +1234567890 - This will send MMS to +1234567890.

    • MMS to +1234567890 - This will send MMS to +1234567890.

    • MMS from SIM 2 to +1234567890 – This will send MMS to +1234567890 using SIM 2. You can check out the ID of the SIM you want to use from the SIMs page.

    • MMS from SenderID 5 to +1234567890 – This will send MMS to +1234567890 using Sender ID 5. You can check out the ID of the Sender ID you want to use from the Sender IDs page.

You have to email the email address shown on this page from your account email after you enable this setting. Your message will be sent up to 1 min late after you send the email. Users can only enable this if it is enabled by Admin.

🆔 Identify Phone Number

Allows automatic identification of phone numbers using contact fields. Shows up information stored in contact fields instead of just displaying the phone number.

  • Enabled: Allows automatic identification of phone numbers only if this option is enabled.

This can significantly slow down loading of messages if there are a large number of messages and contacts associated with your user account. Use with caution.

  • Contact Field Tag: The contact field tag to use for identifying phone numbers. For example, if you have received a message from +1234567890 and there is a contact in the contact list having this number with the name field set to John Doe, then it will show John Doe along with the phone number on the messages page when you have set it to name. It defaults to the value set by Admin.

🆘 Prompts

Settings to add or update keywords used to trigger an action via a received message. You can disable any of the keywords by unchecking the checkbox besides it.

  • Blacklist: Keyword to add number to blacklist.

  • Whitelist: Keyword to remove a number from the blacklist.

  • Subscribe: Keyword to resubscribe the number to the contact list.

  • Unsubscribe Keyword to unsubscribe number from the contact list.

  • Notify: If enabled, it will send an SMS notification to the user when a keyword is triggered.

#️⃣ QR Code

Settings to manage QR code behavior.

  • Lifespan (Seconds): Number of seconds QR code should be considered valid. This will be calculated from the time when the QR code was first generated. You can set it to unlimited by unchecking the checkbox.
