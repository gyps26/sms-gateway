User Help

Send Messages

Click on Send from the navigation menu to start sending messages. You will find a description for every field from below.

Fields

Name

Name of the campaign. It is an optional field, but it will help you keep track of the campaign on the Campaigns list.

Type

Type of message you want to send. It can be SMS, MMS, or WhatsApp.

Sender IDs

Sender IDs you want to use for the campaign. It can be used in conjunction with SIMs. It will split Messages equally if multiple SIMs or Sender IDs are selected.

Sims

SIMs you want to use for the campaign. It can be used in conjunction with Sender IDs. It will split Messages equally if multiple SIMs or Sender IDs are selected.

Recipients

Source of the mobile numbers. It can be one of the following.

  • Enter Manually: Lets you enter the mobile numbers manually one per line.

  • From Spreadsheet: Lets you send messages using mobile numbers from XLS, XLSX, ODS, or CSV file. The file must include a column with the header Mobile Number, listing all the phone numbers you wish to send messages to. You can also use data from other columns, but each column must have a header.

    For example, if you have a CSV file with three columns having header named Mobile Number, First Name and Last Name, then you can use the following syntax to use the data from other columns.

    Hi {first_name} {last_name}, how are you?

    In case of missing data in the row for the above columns used in the message, it will send the message as it is with placeholder like {first_name} intact.

  • Contact Lists: Allows you to select contact lists for sending messages. Any of the contact lists you choose must have at least one subscribed contact. You can also use data from other fields of a contact list in the message.

    For example, if you have fields having first_name and last_name as tag in your contact list, then you can use the following syntax to use the data from them.

    Hi {first_name} {last_name}, how are you?

    If you want to use data from other fields in the contact list while using multiple contact lists, then make sure all the contact lists contain fields you are using in the message. Otherwise, messages will be sent as it is with a placeholder like {first_name} intact.

Template

Let you choose the template you want to use for this message.

Message

Message you want to send in this campaign. You can randomize the message content by using spintax as shown below.

{Hi|Hello|Hey}, How are you? {Best|Kind|Warm} regards, My Name

Delay

This option doesn't affect the messages sent by Sender IDs. Delay between each message when sending to multiple mobile numbers. It should be between 0 and 120. You can also use a range delay like 10–60, and it will select a random delay between 10 and 60 for each message.

Delivery Report

This option doesn't affect the messages sent by Sender IDs. Turn on delivery reports so you can get Delivered status for messages. Implementation of delivery reports vary from operator to operator. Many operators send delivery reports even if a message is received only by the SMS center near the receiver and not the receiver itself. It will not guarantee that the receiver got the message in those cases.

Prioritize

This option doesn't affect the messages sent by Sender IDs. Allows you to choose if you want to send the campaign in a prioritized queue. If you select Yes, then it will postpone any non-prioritized campaign currently running and send this campaign first.

Schedule

Date and Time when you want to send this campaign.

Recurring

Let you choose if you want to repeat the campaign after a certain time frame or not.

Frequency

Only available when you choose to repeat the campaign. It allows you to specify the frequency of the recurring campaign. For example, if you select 1 Month, then it will repeat the campaign every month.

Ends At

Only available when you choose to repeat the campaign. Set the date and time when you want to stop repeating the campaign.

Active Hours

Time of the day when messages should be sent. If there are still messages left to send in the campaign after this time, then it will schedule those messages for the next available day and time.

Timezone

Timezone used by Active Hours, Days of Week, Schedule, and Ends At.

Days of Week

Days of the week when the campaign should be active. If the current day doesn't fall within the selected days, then it will schedule remaining messages for the next available day and time.
